- 项目需求说明书/任务书 Cahier des charges 侧重需求和约束条件
    - 预期的可交付成果
    - 项目目标
    - 项目预算

        具体人员名单属于人力资源计划或项目团队名录

- 利益相关者是指任何可能影响项目或受项目影响的个人、群体或组织，所以不一定是项目内人员、也可能来自外部
    从而降低风险，管理多方期望

- 定义项目任务是属于 范围管理Scope Management 或 WBS工作分解结构的范畴

- PBS: Product Breakdown Structure 产品分解结构是项目管理中“基于产品的规划”技术的重要组成部分
    PBS是一种层级结构，将项目最终要交付的产品、成果物分解为更小的、更易于管理的组件，逻辑是“为了得到最终的产品，需要哪些物理零件或中间产物”
    相比之下，WBS更像是How，而PBS是For What

- 在项目澄清阶段中，需求定义的主要目标是什么
    准确识别并记录项目必须实现的成果以及利益相关者的期望
    - 对齐目标: 确保项目所有人员对成果的定义达成共识
    - 范围控制: 只有定义了需求，才能确定项目 范围 Scope ，防止后期出现无休止的 需求蔓延 Scope Creep

- 项目资源管理的核心在于 优化 与 对齐
    1. 资源获取: 确保在项目的各个阶段，所需的特定技能(人)或设备(物)能够到位，避免因资源短缺导致进度延误
    2. 效率最大化: 仅仅拥有资源不够，还需要合理分配，防止资源闲置或资源过度使用（导致员工倦怠或设备损坏）
    3. 目标导向: 服务于项目的整体目标（质量、时间和成本的平衡）
        例如软开中一般先用普适的技术栈进行快速开发，然后在后期视情况进行性能优化
    - 根据布鲁克斯法则，向进度落后的项目增加人手可能会使其更加落后（例如沟通成本等等的影响）
    - 资源管理是关于预算的合理利用，不能盲目削减预算
    - 资源管理涵盖所有类型的任务，包括行政、管理、沟通等，而不仅仅是技术任务

- 关键资源："关键" - 瓶颈效应、时间约束
    缺乏灵活性: 一般100%占用，没有任何缓冲时间
    直接影响工程终期

- 关键路径是由关键任务连成的（一条或多条）- 2
    关键任务是指只要延误就会影响工程总工期的，也就是总时差为0的

- 哪种工具常用于可视化人力资源在不同任务上的分配，并识别潜在的冲突
    可视化-甘特图Gantt
        甘特图通过时间轴展示任务，清晰地标出每项任务由谁负责，点那个一个人在同一段时间被分配了两个重叠的任务时，甘特图能直观地显示出资源过度分配的冲突

- “资源平衡”nivellement 与 “资源平滑”lissage 的主要区别是什么
    - 资源平滑 - 时间受限
        旨在遵守项目截止日期且不超出资源闲置
        核心：完工日期（关键路径）是第一位的，不能变
        仅在任务的浮动时间范围内调整任务，使得资源的需求量尽可能平稳，避免大起大落
        项目的完工日期不会变、关键路径不会变，无法解决所有资源冲突，因为它受限于浮动时间
        （浮动时间
            - 总时差 Total Float TF
            - 自由时差 Free Float FF
        ）
    - 资源平衡 - 资源受限
        是在重新分配现有资源避免超负荷
        核心逻辑是“资源是第一位的”，如果需求超过了供应
        结果虽然确保了任何时间资源需求都不会超过上限，但是会导致项目关键路径变化，通常会推迟项目的完工日期

- 优先级
    1. 工期 Duree 是第一优先级
    2. 资源利用率 Taux
    3. 工作量 Charge

工期 = 工作量 / 资源利用率

- 当用户修改一个变量时，在不同的 fixe 固定类型下系统会自动调整哪一个来保持上面式子平衡
    当 Charge-Taux fixes 也就是固定工作量和利用率的时候，如果修改了 Duree ，那么就得不得不优先调整 Taux 利用率
    这是因为题目信息 Charge-Taux 的顺序表明了 Taux 的优先级比 Charge 低，如果是 Taux-Charge 的话就是 Taux 优先级高、先动 Charge

# 3 财务

## 3.1 计算与分析项目盈利性

### 基本概念

- 盈亏平衡点 SR ：企业收支平衡的最低营业额
- 营业额 CA ：企业年度总销售额
- 费用
    - 固定费用 CF - Fixes ：不衰活动水平变化的费用，例如租金、薪资、保险等等
    - 可变费用 CV - Variables ：随生产量成比例变化的费用，例如原材料、部分能源

### 计算步骤
1. 确定固定费用
2. 确定可变费用
3. 确定历史营业额去年的 CA
4. 可变成本利润：MCV = CA - CV
5. 可变成本利润率：TMCV = MCV / CA
6. 盈亏平衡点 SR = CF / TMCV
7. 死点天数 PM = SR / (年度CA/360) // 达到平衡所需的天数
    死点数量 = SR / 产品单价 // 达到平衡的所卖产品数

### 注意点
- 帮助评估绩效、设定销售目标、优化成本（例如降低CF以缩小SR）
- 有局限性，因为是基于的往年数据CA，适用于稳定业务不适合季节性波动的
- 在商业计划中要先算 SR 再构建完整财务模型

## 3.2 主要财务文件
- 损益表
    用于反映公司在特定时期内的经营成果（利润或亏损）
    - 收入：公司通过销售商品或提供服务获得的收入
    - 费用：为产生收入而发生的各项支出
    => 盈利利润、净利润
    项目盈利率：投入资本与产出利润之间的比例
- 资产负债表
    - 资产：公司拥有的资源
    - 负债：公司的资金来源，包括自有资金和债务
通过财务文件对创业进行数字化管理，从而实现收入 Produits 覆盖成本 Charges 的盈利的目的
损益表 - 一段时间
    是动态的，记录一段时间内的流入与流出
资产负债表 - 某一时刻
    是静态的，表示某一特定时刻公司的财务状况

## 3.3 融资
financement 为确保项目在整个生命周期内顺利实施而筹集、分配和管理所需资金的过程
1. 钱从哪里来 - 资金来源
2. 什么时候需要 - 现金流时间安排
3. 如何使用与控制 - 预算和成本控制

- 公司资金来源
    - 自由资金：创业者的个人出资、合伙人投资、留存收益
    - 外部融资：
        - 银行贷款
        - 天使投资或风险投资
        - 政府补贴
- 融资策略
    1. 明确需求：融资前需精确计算所需的设备投资和营运资金
    2. 盈利能力证明：损益表 向出资方证明项目具备盈利潜力的关键
    3. 可持续性：合理的资产负债结构和稳健的现金流规划

- 资产负债表反映企业在会计期结束的财务状况 - 企业的家底
    资产负债表 Bilan 是某一时刻的快照

- 企业的财产不是利润
    财产 = 资产 - 负债
    而利润是一个期间内的经营成果

- Produits − Charges = Résultat
    收入 - 支出 = 结果（盈利或亏损）

- 企业欠供应商的是“负债” Passif
    企业拥有或控制的资源是叫资产 Actif

- 增加库存 -> 花钱买东西 -> 现金减少、会恶化现金流

- 折旧反映固定资产的使用和损耗
    折旧 = 把固定资产成本分摊到使用年限
        是一种非现金的费用

- 盈亏平衡点是开始盈利所需实现的营业额

- 营业毛盈余反映企业的经营绩效
    EBE excedent brut d'exploitation 衡量主营经营能力
    不受折旧、税费、融资影响

- 负债是企业欠他人的东西
    包括 借款、应付账款、税款 等

- 通过乡银行借款可以改善营运资金

- 异常费用：非经常性的（“每年发生”这种词肯定就错了）
    如：罚款、资产报废

- 即使营业毛盈余为负，也可能实现利润
    如大额金融收益
    异常收益等等
    利润不只来自经营活动！

- 资本 = 股东投入
    本年度赚的钱 = 利润

- 利润计算得用利润表
    - 资产负债表是存量，而不是看流量的

- 给客户更长付款期限可以改善现金流这是错误的
    因为晚付款意味着现金更晚到账

- 预计费用是不确定但计入损益的费用
    - 金额或发生时间不确定
    但是风险已经存在

- 企业欠他人的部分属于其财产结构的一部分
    - 财产结构 = 资产 + 负债

- 企业的机器设备列在资产负债表的资产中
    机器 = 固定资产

- 通过获得供应商延期付款可以改善现金流
    注意：是“供应商”-延期
    那么就是我们的钱可以晚点付，仍在企业里面，非常有利于现金流

## 3. 风险与机会分析
1. 基本定义
    - 风险：不确定性对实现目标产生的影响
    - 风险源：单独或共同具有产生风险内在潜力的要素
    - 不确定性：由于事件及其后果的信息缺乏了解而导致的状态

2. 风险评估流程 - 三个阶段！
    1. 风险识别
    2. 风险分析
    3. 风险评价

3. VIDeC-Project 方法
通过量化定性指标计算风险的严重性/关键性
关键性 = 可能性 * 影响 * 可检测性
- 可能性：事件发生概率
- 影响：对项目目标产生的后果
- 可检测性：在风险产生影响前，发现或预判其发生的能力
决策标准：低只需监视、中需要采取行动、高的话得立即采取行动

4. 风险处理
当风险评估结果为不可接受时，需要采取预防性措施
- 针对风险源
- 针对可能性
- 针对影响
- 针对可检测性

5. 风险制图
RMI = 参数1的影响等级 * 参数2的影响等级
    如 发生概率*影响等级

1. 临时性
2. 阶段性
3. 结果导向
